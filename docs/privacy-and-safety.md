# 私隱及安全守則

本專案是公開展示網站，因此所有顯示在網站上的相簿及相片都應視為公開內容。

## 核心原則

1. 只展示已審核、適合公開的相片。
2. 不展示未經同意或不適合公開的學生相片。
3. 不在網站顯示學生全名、電話、地址、身份證明文件、醫療資料或其他敏感資料。
4. 不把私人 token、API_SECRET、password 或管理員密碼放入 GitHub Pages 前台。
5. 不使用公開前台執行新增、刪除、修改或審核資料功能。

## Google Drive 相片管理

建議將相片分為兩類資料夾：

```text
原始相片資料夾
        ↓ 篩選及審核
網站展示用相片資料夾
```

網站只應連接「網站展示用相片資料夾」。

## Google Sheet 管理

Google Sheet `Albums` 的 `公開顯示` 欄控制相簿是否在網站顯示。

- 勾選：公開展示。
- 不勾選：不在網站顯示。

`分類` 只控制相簿顯示在 `焦點活動` 或 `活動回顧`，不代表額外權限。

## GitHub Pages 安全限制

GitHub Pages 是公開靜態網站：

- 無內建登入權限。
- 無法安全保存 secret。
- 所有前台 JavaScript 都可被瀏覽者查看。

因此，任何需要保護的管理功能都不應放在 GitHub Pages 前台。

## Apps Script API 安全

目前 Apps Script 只提供公開 read-only API：

- `albums`
- `photos`
- `latest`
- `list`

這些 action 只讀取已公開相簿資料，不應進行寫入、刪除或修改。

如日後需要後台管理功能，應另建受保護的管理介面，並使用登入或 secret 保護。

## 公開前檢查清單

發布新相簿前請檢查：

1. 相片是否已審核。
2. 是否不含敏感個人資料。
3. Google Drive folder 是否是網站展示用資料夾。
4. Google Sheet `公開顯示` 是否正確。
5. 相簿簡介是否避免使用學生全名或敏感描述。
6. 封面圖片是否適合公開展示。
